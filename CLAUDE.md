# CLAUDE.md — GiottoClass

## Package Overview

**GiottoClass** is the foundational S4 object framework for the [Giotto Suite](https://drieslab.github.io/GiottoClass/) of spatial omics analysis tools. It defines the `giotto` object and all its subobjects, data access methods, and basic object creation utilities. Other packages in the suite (GiottoVisuals, GiottoAnalysis, etc.) build on top of the classes defined here.

- **Package version:** 0.5.0
- **R requirement:** >= 4.5.0
- **Primary spatial backend:** `terra` (SpatVector / SpatRaster)

## S4 Class Hierarchy

```
giottoSubobject (virtual base)
├── exprData          → exprObj
├── coordDataDT       → spatLocsObj, cellMetaObj, featMetaObj
├── spatData          → giottoPolygon, giottoPoints, giottoBinPoints
├── dimObj
├── spatialNetworkObj, spatialGridObj
├── spatEnrObj, nnNetObj
├── giottoImage, giottoLargeImage, giottoAffineImage
└── overlapPointDT, overlapIntensityDT

giotto (main object, 22 slots — lists of the above subobjects)
```

The `giotto` slot lists are keyed by `spat_unit` (spatial unit, e.g. `"cell"`) and `feat_type` (feature type, e.g. `"rna"`).

## Coding Conventions

Full style guide: https://giottosuite.com/articles/coding_style.html
Dispatch functions: https://giottosuite.com/articles/dispatch_functions.html

### Naming

| Type | Convention | Example |
|------|-----------|---------|
| Exported user-facing functions | `camelCase` | `createGiottoObject()` |
| S4 classes | `camelCase` | `giottoLargeImage` |
| Exported utility / non-user-facing | `snake_case` | `dt_to_matrix()` |
| Internal functions | `.snake_case` (leading dot) | `.detect_in_dir()` |
| Parameters / arguments | `snake_case` | `spat_unit`, `feat_type` |

Do not use periods as name separators — this conflicts with S3 dispatch.
Do not use non-UTF-8 characters; use Unicode escapes instead (e.g. `\u00F6`).

### Other

- **Style:** 4-space indentation, following BioConductor conventions (`biocthis` + `styler`). Enforced by `lintr` in CI.
- **S4 patterns:**
  - Generic definitions go in `R/generics.R` with `@rdname` tags.
  - Method implementations go in the corresponding `R/methods-<topic>.R` file.
  - New slot types may need class union updates in `R/zzz.R`.
- **Imports:** Add all `@importFrom` declarations to `R/package_imports.R`, not scattered across files.
- **Error handling:** Use `GiottoUtils` helpers (`gstop()`, `gwarn()`, `gmessage()`) rather than bare `stop()`/`warning()`.
- **Validation:** Use `checkmate` for input validation at function boundaries.
- **Data tables:** The package uses `data.table` throughout; prefer `data.table` idioms over base R for tabular operations.

## Adding New Classes

1. Define the class with `setClass()` in the appropriate `R/classes-*.R` file.
2. Export it via `@exportClass` in its roxygen block.
3. Define a constructor (e.g., `createGiottoFoo()`) in `R/create.R`.
4. Add any required generics to `R/generics.R`.
5. Add method implementations in a new or existing `R/methods-*.R` file.
6. If the class needs to be stored inside a `giotto` object slot, update the slot type union in `R/classes.R` and `R/zzz.R`.
7. Add a `packed*` variant if the class wraps a `terra` spatial object (needed for serialisation).
8. Write tests in `tests/testthat/`.

## Testing

```r
# Run all tests
devtools::test()

# Run a specific file
testthat::test_file("tests/testthat/test_03_accessors.R")
```

**Test data** is loaded from the `GiottoData` package via helpers like `GiottoData::loadGiottoMini()` and `GiottoData::loadSubObjectMini()`. Global test setup lives in `tests/testthat/setup.R`, which sets:

```r
options("giotto.use_conda" = FALSE)      # No conda during tests
options("giotto.no_python_warn" = TRUE)  # Suppress Python warnings
options("lifecycle_verbosity" = "quiet")
```

Tests are numbered (`test_01_*.R` … `test_10_*.R`) for ordered execution of foundational object creation, then named for functional areas.

## Dependencies

**Hard imports (always required):** `terra`, `data.table`, `Matrix`, `igraph`, `dbscan`, `deldir`, `checkmate`, `matrixStats`, `reticulate`, `GiottoUtils`

**Optional (Suggests):** `Seurat`/`SeuratObject`, `SingleCellExperiment`, `SpatialExperiment`, `sf`, `sp`, `stars`, `magick`, `HDF5Array`, `qs`, and others — check `DESCRIPTION` for the full list.

When adding a new dependency:
- Hard requirement → add to `Imports:` in `DESCRIPTION` and add `@importFrom` in `R/package_imports.R`.
- Optional → add to `Suggests:` and guard usage with `package_check("pkg", repository = "CRAN")` (or `"Bioc"` / `"github"`).
